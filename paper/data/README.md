# Data provenance

The worked example uses two files distributed with CSUBST and associated with
Fukushima and Pollock (2023), DOI
[`10.1038/s41559-022-01932-7`](https://doi.org/10.1038/s41559-022-01932-7).

## Immutable source

- Repository: <https://github.com/kfuku52/csubst>
- Source commit: `1b0d98eec71702538e2c09fef60ff1d50dc7e2f4`
- Repository license at that commit: MIT License, copyright Kenji Fukushima
  (2017)

|---|---|---|
| `pepc/PEPC.tree.nwk` | `csubst/dataset/PEPC.tree.nwk` | `fe287d9ce9802fa35e6707cb33f187d96f06e0e55d54aadde27348993df7c8f4` |
| `pepc/PEPC.foreground.tsv` | `csubst/dataset/PEPC.foreground.tsv` | `51a46eb5634cf8077303a6514536a38b6c1e1d4f0237a6d739449e8074528227` |

`paper/analysis/worked_example.py` downloads the files from raw GitHub URLs at
the pinned commit when they are absent and refuses to continue if either
checksum differs.

## Transformations

The Newick tree is retained byte for byte. Foreground regular expressions are
expanded deterministically to one C3/C4 row per tree tip and written to
`paper/results/pepc/pepc_traits.tsv`. Subsequent files in that result directory
are produced by the NWKIT validation, monophyly, ancestral-state, sampling, and
drawing commands. No manual tree editing is performed.

## Deposit destination

The two inputs, their derived result files, and the figure-ready summaries are
included in the manuscript Dryad package generated by
`paper/analysis/build_archives.py`. The NWKIT source and analysis scripts are
included separately in the Zenodo code package.
